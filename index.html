Re-Learning Git and GitHub

Common Commands:
  • git clone: This is used to make a copy of your repository on Git Hub available on your local machine.
  • git add: When new changes are made, deleted or new files are made, you use this command to have git track those changes.
  • git commit: Saves your changes
  • git push: This will upload your changes to a remote repository on GitHub
  • git pull: When changes have been made to the remote repository you want to use the “pull” command to download those changes.
  • git status: Shares what changes have been made but are not being tracked
  • git branch: Lists the branches you have on the repository and then puts an asterix next to the branch you are currently on.
  • git checkout -b "branch name": Creates a new branch.
  • git diff "branch name It compares 2 versions of the code and shows what lines have been changed
  • git push -u origin "branch name"/ git push --set-upstream origin "branch name": Allows you to push your changes to your newly
    created branch which you can then PR into a master branch.
  • git merge "branch name":Updates the branch you are on with the changes in the branch "branch name".
  • git reset "file name": Unstages the file.

    • Note:

    HEAD in git is a pointer to the last commit.

  • git reset HEAD~1: Git will undo last commit made. Pointer will now be on commit before most recent commit.
  • git log: Lists all commits in reverse order. Lists from latest to oldest commits.
  • git reset "commit hash": Unstages any changes that were made after that commit hash.
  • fork: Usually for if you don't have access to a repo and you would like to be able to branch and PR into it.
